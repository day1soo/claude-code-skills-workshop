## Step 2-1. Plan Mode로 나만의 스킬 설계

### EXPLAIN

#### 왜 나만의 스킬을 만드나?
- **Monday-Checklist 체험**: "이건 Claude Code에서만 가능하네!" 차별화 확인
- **개인 맞춤 자동화**: 내 업무 패턴에 딱 맞는 스킬 제작
- **반복업무 자동화**: 구체적인 시간 절약 효과 측정

#### Plan Mode란?
Claude와 함께 **체계적으로 스킬을 설계하는 모드**입니다.
- 요구사항 → 설계 → 구현 단계적 진행
- 놓치는 부분 없이 완성도 높은 스킬 제작

### EXECUTE

**1단계**: Plan Mode 진입
```
plan mode로 [내 반복업무]를 자동화하는 스킬을 만들어줘
```

**2단계**: 요구사항 정리
```
Plan Mode 질문들:
- 어떤 반복업무를 자동화하고 싶나요?
- 현재 소요시간은 얼마나 되나요?
- 어떤 데이터 소스가 필요한가요? (Gmail/Slack/Calendar/기타)
- 최종 결과물 형태는 어떤가요?
```

**3단계**: 스킬 제작 실행
- Plan Mode에서 설계된 내용으로 실제 스킬 제작
- MCP 연동 및 자동화 로직 구현
- 테스트 및 개선

---
👆 위를 직접 해보시고, 끝나면 "다음"이라고 입력해주세요.

## Step 2-2. GitHub으로 팀 공유하기

### EXPLAIN

#### 왜 GitHub으로 공유하나?
**개인 사용 → 팀 확산 → 조직 문화 변화**

**GitHub 공유 장점**:
- 버전 관리 가능
- 팀원들이 `git clone`으로 쉽게 다운로드
- 이슈/피드백 체계적 관리
- "진짜 개발팀처럼" 전문적 공유

### EXECUTE

#### 2가지 공유 방법 중 선택

**방법 1**: GitHub 자동 업로드 (추천: 팀 단위)
```
GitHub에 [스킬이름] 업로드해줘
```
Claude가 자동으로:
- 새 리포지토리 생성
- SKILL.md + README.md 업로드  
- 팀원 공유용 링크 제공

**방법 2**: SKILL.md 파일 직접 공유 (추천: 간단한 공유)
```
[스킬이름] 폴더를 압축해서 Slack에 공유
또는
SKILL.md 내용을 복사해서 Notion 페이지 생성
```

---
👆 위 중 하나를 선택해서 실행하시고, 끝나면 "다음"이라고 입력해주세요.

## Step 2-3. 실제 공유 및 상호 체험

### EXPLAIN

#### 상호 체험의 가치
- **다양한 스킬 아이디어** 발견 (다른 팀원들의 반복업무 해결법)
- **실제 사용성** 검증 (다른 사람도 쉽게 쓸 수 있나?)
- **GitHub/파일 공유** 실습으로 전문적 협업 경험

### EXECUTE

**1단계**: GitHub 링크 또는 파일 교환
- 팀원들과 GitHub 리포지토리 링크 교환
- 또는 SKILL.md 파일 Slack으로 공유

**2단계**: 빠른 상호 체험
```
팀원 스킬 간단 실행:
"[팀원이름]님 스킬대로 [업무] 자동화해줘"
```

**체크 포인트**:
- "이 아이디어 우리 팀에도 적용하면 좋겠네요!"
- "GitHub으로 공유하니까 진짜 개발자 된 느낌!"
- "내일부터 실제로 이 스킬들 써볼게요"
