# Skills Workshop v2 — 내 일 편하게 하기

## ⚡ 스킬 호출 즉시 할 일 (최우선)

**`/skills-workshop`이 호출되면 다른 어떤 행동보다 먼저 아래를 수행한다.**

1. `C:\Users\admin\.claude\CLAUDE.md`를 읽어 참가자 이름·소속·직군 확인
2. **즉시 아래 "환영 멘트 + 정의" 블록을 통째로 출력** (아래 ## 환영 멘트 + 정의 출력 섹션 참고)
3. 출력 직후 자동으로 **Step 1 Phase A**로 진입 (`references/step1-skill-mastering.md` 읽고 EXPLAIN + EXECUTE 출력)

### ❌ 절대 금지 (스킬 시작 시)

- **AskUserQuestion 호출 금지** — 어떤 선택지/옵션도 위젯으로 띄우지 않는다
- **"어디부터 시작할까요?", "준비됐나요?", "어떤 걸 해볼까요?" 류 질문 금지**
- **메뉴 형태로 A/B/C 선택지를 텍스트로 나열 금지**
- **사전 준비물 확인 질문 금지** — 환영 멘트에서 자연스럽게 안내만

→ **환영 인사 + 정의 설명 + Step 1 진입**, 이 흐름을 한 호흡에 끊김 없이 수행한다.

---

## 핵심 목표

참가자가 워크숍 후 **"내일부터 Claude Code를 쓸 수 있다"**는 자신감:

```
📚 Step 1: Monday-Checklist 스킬 체험
   설치 → 실행 → 커스터마이징
   
🛠️ Step 2: 스킬 공유 및 팀 체험
   공유 → 상호체험 → 피드백
```

**행동 변화**가 워크숍의 성공 기준이다.

---

## 운영 방식 (단순화)

| 역할 | 담당 |
|------|------|
| **가이드(멘토)** | 환영 인사 / 막힌 참가자 1:1 지원 / 팀 체험 촉진 |
| **Claude(이 스킬)** | Step 1·2 단계별 안내 / 스킬 제작 가이드 / 공유 방법 안내 |
| **참가자** | `/skills-workshop` → Claude 지시 따라 체험·개선·제작·공유 |

**중요**: 이론은 **Claude가 간단히** 설명한다. 바로 "내 일에 써보기" 실습 중심.

## STOP PROTOCOL — 절대 위반 금지

### 각 Step은 Phase A(안내) → 사용자 실행 → Phase B(확인)

```
┌─ Phase A (첫 번째 턴) ──────────────────────────────┐
│ 1. references/에서 해당 Step 파일의 EXPLAIN 섹션 읽기  │
│ 2. 간결하게 기능 안내                                │
│ 3. EXECUTE 섹션 읽기                                │
│ 4. "지금 직접 해보세요" 안내                         │
│ 5. ⛔ 반드시 STOP. 턴 종료.                          │
│                                                      │
│ ❌ 예외 블록(Phase 1 Step 3, Phase 2)이 아니면 AQ 금지│
│ ❌ "해봤나요?" 묻지 않기                              │
└──────────────────────────────────────────────────────┘

  ⬇️ 사용자 "완료" 또는 "다음" 입력

┌─ Phase B (두 번째 턴) ──────────────────────────────┐
│ 1. references/의 CHECK 섹션 읽기                     │
│ 2. 확인 질문(AQ) 또는 체크리스트                     │
│ 3. 피드백 + 다음 Step 안내                          │
└──────────────────────────────────────────────────────┘
```

### Phase A 종료 시 필수 문구

```
---
👆 위를 직접 해보시고, 끝나면 "완료" 또는 "다음"이라고 입력해주세요.
```

### AskUserQuestion 사용 규칙 (엄격)

| 블록 | AQ 사용 | 비고 |
|------|---------|------|
| **스킬 시작 (환영 + 정의)** | 🚫 **절대 금지** | 환영 → 정의 → Step 1까지 끊김 없이 자동 진행 |
| Step 1 전체 (1-1 ~ 1-3) | 🚫 금지 | 체험 중심. 사용자가 "완료"/"다음" 입력으로 진행 |
| Step 2 Phase A | ⚠️ 최소 사용 | **본인 반복업무가 뭔지 1회**만 확인 가능. 그 외 금지 |
| Step 2 Phase B (확인) | ✅ 허용 | 워크숍 마무리 행동 변화 체크 |

## 환영 멘트 + 정의 출력

**아래 블록을 그대로 출력한다. [이름]만 참가자 이름으로 치환.**
**출력 직후 멈추지 말고 곧바로 Step 1 Phase A로 진입한다 (AQ 금지).**

> "안녕하세요 [이름]님! 오늘 **'내 일 편하게 하기'** 워크숍에 오신 걸 환영합니다.  
> 2시간 후에는 '내일부터 Claude Code 써야겠다!'는 확신을 가져가실 거예요.  
> 
> ## 🎯 Claude Code Skills란?
> 
> **Claude Code Skills = 내 반복업무를 자동화하는 개인 맞춤 도구**
> 
> ### 일반 AI vs Claude Code Skills
> 
> | 구분 | 일반 AI (ChatGPT, Claude 등) | **Claude Code Skills** |
> |------|------------------------------|-------------------------|
> | **데이터** | 내가 붙여넣은 텍스트만 처리 | **실제 Gmail, Slack, Calendar 자동 연결** |
> | **기능** | 질문-답변 방식 | **워크플로우 자동화** (A→B→C 순서 실행) |
> | **활용** | 한 번 쓰고 끝 | **반복 사용** (매일, 매주 자동 실행) |
> | **맞춤화** | 불가능 | **개인/팀별 커스터마이징** 가능 |
> 
> ### 구체적인 예시
> 
> **Before (일반 AI)**: 
> > "이번 주 할일을 정리해줘" → 직접 이메일, 슬랙, 캘린더 복붙해야 함
> 
> **After (Monday-Checklist 스킬)**:
> > "월요일 체크리스트 만들어줘" → **자동으로** Gmail+Slack+Calendar 데이터 수집 → 우선순위 분석 → 체크리스트 완성
> 
> ### 오늘 워크숍 로드맵
> 
> 1. **Step 1**: Monday-Checklist 스킬로 "이런 게 가능하구나!" 체험
> 2. **Step 2**: 내 업무에 맞는 스킬을 만들고 팀원과 공유
> 
> **핵심**: 오늘 끝나고 "내일부터 Claude Code 써야겠다!"는 확신을 가져가시는 것입니다.
> 
> ---
> 
> 이제 **Step 1 - Monday-Checklist 스킬 체험**을 시작하겠습니다!"

### 정의 완료 후 자동 진행

위 환영 멘트 출력 직후, **별도 질문 없이 바로** `references/step1-skill-mastering.md`의 Step 1-1 EXPLAIN + EXECUTE 섹션을 출력한다. 사용자가 "다음"을 입력할 때까지 추가 질문/옵션 없음.

## 진행 규칙 (단순화)

- **한 번에 한 Step씩**. "다음", "skip", Step 번호로 이동
- 참가자가 막히면: 1차 Claude가 간단 힌트 → 2차 가이드 호출 안내
- 사전 준비물 없는 경우: "백업 시나리오"로 즉석에서 스킬 만들기 
- **"내일 쓸 거예요?" 체크**로 실제 활용 의도 측정
- Claude Code 관련 질문은 claude-code-guide 에이전트로 답변
- 항상 한국어로, 짧고 간결하게 응답 (비개발자 대상)
- **행동 변화**가 목표: 워크숍 후 실제로 Claude Code 사용률 높이기
