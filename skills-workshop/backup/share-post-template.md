# Slack 공유 게시물 템플릿

Phase 2 Step 2에서 참가자가 작성할 Slack `#claude-skills` 게시물 포맷.

---

## 기본 템플릿

```
[스킬 이름] — [한 줄 소개]

🎯 무엇을 하는가
(1~2줄로 기능 요약)

👤 이런 분에게 유용
(직군 / 상황 / 역할)

📥 이런 Input을 받으면
(Input 형태 — 텍스트 / 파일 / URL 등)

📤 이런 Output을 냅니다
(Output 형태 — 표 / 리스트 / 문서 등)

⚠️ 주의사항
(있다면 / 없으면 "없음")

📂 설치 / 사용
원본 Claude 터미널에서:
/skills add [my-스킬이름]

또는 첨부 SKILL.md 파일 다운로드 후
~/.claude/skills/[my-스킬이름]/SKILL.md 위치에 저장

---
[내 이름 / 소속]
v0.1 — [작성 날짜]
```

---

## 작성 예시 1 (주간보고 스킬)

```
my-weekly-report — 주간 업무 메모를 정식 주간보고서로 변환

🎯 무엇을 하는가
한 주 동안의 업무 메모(완료/진행/블로커)를 받아서 
"[이번 주 완료 / 진행 중 / 다음 주 계획 / 이슈]" 4섹션 보고서로 자동 정리

👤 이런 분에게 유용
매주 월요일 팀 보고서 써야 하는 HR/운영 직군
"뭐 했는지 기억 안 나서 매번 처음부터 쓰는 분"

📥 이런 Input을 받으면
- 한 주 업무 메모 (Slack 요약, 캘린더, 본인 노트)
- 자유 형식 텍스트 OK

📤 이런 Output을 냅니다
- 4섹션 보고서 Markdown
- 섹션별 3~5줄 이내 압축

⚠️ 주의사항
- 회사 공식 템플릿과 다를 수 있음 (본인 확인 필요)
- 이슈 섹션은 "없음" 기본값 — 수동으로 바꿔야 함

📂 설치 / 사용
/skills add my-weekly-report
또는 SKILL.md 파일 첨부 참고

---
곽연수 / HR본부 HR팀
v0.1 — 2026-04-22
```

## 작성 예시 2 (가공본 공유)

기존 스킬을 가공한 버전 공유 시 **원본 출처 명시** 필수:

```
my-content-digest-ko — content-digest를 한국어 전용으로 가공

🎯 무엇을 하는가
원본 content-digest가 영어로 요약할 때가 있어서, 
**항상 한국어로만** 요약하도록 body 제약 추가

👤 이런 분에게 유용
한국어 콘텐츠를 주로 다루는 모든 직군

📥 이런 Input을 받으면
YouTube URL / X(트위터) URL / 웹페이지 URL / PDF 텍스트

📤 이런 Output을 냅니다
한국어로만 된 요약 (존댓말)

⚠️ 주의사항
- 원본: content-digest (공개 스킬)
- 가공 포인트: 언어 강제 = "한국어", 문체 = "존댓말"

📂 설치 / 사용
SKILL.md 첨부 — ~/.claude/skills/my-content-digest-ko/SKILL.md

---
곽연수 / HR본부 HR팀
v0.1 — 2026-04-22
원본 출처: content-digest (claude.ai/skills)
```

---

## 작성 팁

1. **한 줄 소개는 20자 이내** — 스레드 목록에서 눈에 들어와야
2. **"이런 분에게 유용"에 직군 최소 2개** — 본인만 쓸 게 아니라면
3. **주의사항은 솔직하게** — 숨기면 오히려 나쁨. "v0.1이라 덜 다듬어짐" 같은 표기 OK
4. **버전(v0.1)은 꼭 쓰기** — 개선 버전 업로드 시 추적 가능
5. **가공본이면 반드시 원본 출처 표시** — 오픈소스 예절

## 운영자용 — 첫 게시물 대응

참가자가 Slack에 처음 올린 게시물에는:
- 🙌 이모지 달기 (30분 안에)
- "첫 공유 환영합니다!" 댓글 (하루 안에)
- 다른 참가자 중 이 스킬 쓸 만한 사람 태그

이것만으로 공유 문화가 유지됩니다.
