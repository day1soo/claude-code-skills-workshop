# Phase 1 데모 스킬 후보 (강사용)

Phase 1 Step 1에서 참가자가 함께 설치·해부·가공할 **공통 데모 스킬** 3개 후보.
강사가 **당일 아침 참가자 직군 분포**를 보고 1개 선정.

---

## 선정 기준

| 기준 | 설명 |
|---|---|
| **직군 보편성** | 영업·마케팅·운영·HR 모두 공감할 수 있는가 |
| **감탄 포인트** | "어 이게 되네?"가 자연스럽게 나오는가 |
| **가공 여지** | description·body를 바꿔볼 여지가 충분한가 |
| **복잡도** | 비개발자가 10분 안에 해부 가능한가 |

---

## 후보 1 — `content-digest` 🎬

**한 줄**: YouTube 영상/PDF/트윗/웹페이지를 요약해주는 스킬

**장점**:
- ✅ 감탄 포인트 최고 — "YouTube URL만 넣었는데 요약이 되네?"
- ✅ 모든 직군 공감 (리서치·학습·트렌드 파악)
- ✅ 가공 여지 풍부 (언어/형식/깊이 조정)

**단점**:
- Input 형태가 다양해서 해부할 때 조금 복잡

**추천 직군 분포**: 마케팅·영업이 많을 때

**설치 명령**:
```
npx skills add content-digest
```

**샘플 Input 3개 (강사 준비)**:
- YouTube 영상 URL (5~10분짜리 짧은 것)
- PDF 파일 경로
- 트위터/X 게시물 URL

**가공 아이디어 힌트 카드**:
- "한국어로만 요약" (언어 강제)
- "3줄 초과 금지" (분량 강제)
- "HR 관점으로 요약" (도메인 필터)
- "표 대신 불릿으로" (포맷 변환)

---

## 후보 2 — `internal-comms` 🏢

**한 줄**: 사내 상태보고·리더십 업데이트·FAQ·공지 작성 가이드

**장점**:
- ✅ HR·운영 직군에 즉시 유용
- ✅ Output 형식이 명확해서 해부하기 쉬움
- ✅ 가공 여지 풍부 (회사 톤·용어 반영)

**단점**:
- 영업/마케팅에는 공감도 다소 낮을 수 있음

**추천 직군 분포**: HR/운영/백오피스가 많을 때

**설치 명령**:
```
npx skills add internal-comms
```

**샘플 Input (강사 준비)**:
- 주간 팀 업데이트 러프 메모
- 사내 규정 변경 안내 원문

**가공 아이디어 힌트 카드**:
- "데이원컴퍼니 공식 톤으로"
- "직급 호칭 없이 이름+님으로"
- "분량 5문단 이내"
- "영문 용어는 한글 병기"

---

## 후보 3 — `doc-coauthoring` 📝

**한 줄**: 문서를 단계별로 함께 작성해주는 협업 가이드 (제안서·기술 스펙·결정 문서)

**장점**:
- ✅ 비개발자 진입장벽 최저 (대화형)
- ✅ 어떤 직군이든 문서 작업 있음
- ✅ Phase 2 제작 경험과 자연스럽게 이어짐

**단점**:
- 감탄 포인트는 1·2번보다 약함
- 한 번 실행에 시간이 오래 걸릴 수 있음

**추천 직군 분포**: 직군이 매우 다양할 때 / 신입 중심일 때

**설치 명령**:
```
npx skills add doc-coauthoring
```

**샘플 Input (강사 준비)**:
- "신규 프로젝트 제안서를 써야 해" 정도 막연한 시작 지시

**가공 아이디어 힌트 카드**:
- "회사 템플릿 4섹션으로 고정"
- "초안 후 체크리스트 자동 제공"
- "결재 라인 끝단 멘트 추가"
- "영문 혼용 시 한국어 병기"

---

## 당일 선정 결정 트리

```
참가자 직군 분포 확인
│
├─ 마케팅·영업 > 50%
│  └─ 1번 content-digest 선택 (감탄 포인트 중시)
│
├─ HR·운영·백오피스 > 50%
│  └─ 2번 internal-comms 선택 (실용성 중시)
│
└─ 혼합 (뚜렷한 다수 없음) 또는 신입 다수
   └─ 3번 doc-coauthoring 선택 (보편성 중시)
```
