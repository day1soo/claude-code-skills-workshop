# Phase 1 Step 2 — 해부(Reading) (15분)

**목표**: 남의 SKILL.md를 **읽을 수 있는 눈**을 장착. 이 눈이 있어야 가공(Step 3)과 제작(Phase 2)이 가능.

---

## EXPLAIN

### 스킬은 결국 한 장짜리 레시피

스킬은 `.md` 파일 하나예요. 크게 3부분으로 나뉩니다.

```markdown
---
name: content-digest              ← 🟡 이름
description: 콘텐츠를 요약하는 스킬. 
             "요약해줘", "정리해줘" 요청에 발동.  ← 🟡 이름표(트리거 포함)
---

# 콘텐츠 요약                      ← 🟣 제목

... 본문 ...                      ← 🔵 조리 단계
```

- 🟡 **frontmatter (이름표)** — Claude가 "이 스킬 꺼낼지" 판단하는 부분
- 🟣 **제목** — 사람이 읽기 편하도록
- 🔵 **본문 body** — 실제 실행 순서

### body에서 볼 것 4가지

1. **Step이 몇 개인가** (2단계? 5단계?)
2. **Input으로 뭘 받나** (URL? 텍스트? 파일?)
3. **Output으로 뭘 내나** (표? 리스트? 문서?)
4. **제약은 뭔가** ("3줄 이내", "한국어로", 금지 사항 등)

### 가장 중요한 마지막 질문 — "경계 찾기"

> **"이 스킬이 **안 해주는 것**은 뭐지?"**

이게 다음 단계(가공)의 출발점이에요. 경계를 봐야 "여기를 내 상황에 맞게 넓히자/좁히자"가 나옵니다.

## EXECUTE

### Step 2-1. SKILL.md 파일 열기

Claude에게 이렇게 요청하세요 (또는 직접 Read 도구 사용):

```
방금 설치한 [스킬 이름]의 SKILL.md 파일을 읽어줘.
```

Claude가 설치 경로(보통 `C:\Users\admin\.claude\skills\[스킬이름]\SKILL.md`)에서 읽어옵니다.

### Step 2-2. 3분 분석 — frontmatter

화면에 떠 있는 SKILL.md의 **맨 위 `--- ~~~ ---` 사이**를 보세요.
- `name:` 이 뭐죠?
- `description:` 이 한 문장으로 뭘 하는 스킬인가요?
- description 안에 따옴표로 감싸진 **트리거 키워드**가 있나요? 어떤 말들인가요?

### Step 2-3. 7분 분석 — body 4가지 질문

본문을 훑어보면서 아래 4가지를 **각 한 줄로 적어주세요** (머릿속 정리용):

```
1. Step 개수: ___개
2. Input: ___을 받는다
3. Output: ___을 낸다
4. 제약: ___ (없으면 "없음")
```

### Step 2-4. 3분 분석 — 폴더 훑기

스킬 디렉토리에 어떤 폴더가 있는지 보세요 (Bash로 `ls`).
- `references/` — 세부 단계별 자료
- `templates/` — 빈 골격 템플릿
- `samples/` — 예시 input/output

모든 스킬이 폴더를 다 가지진 않아요. **없는 폴더는 그 스킬이 그만큼 단순하다**는 뜻.

### Step 2-5. 2분 — "경계 찾기" 한 줄

가장 중요. SKILL.md를 다시 한 번 훑고, 아래를 한 줄로 적어주세요:

```
이 스킬이 안 해주는 것 = ___
```

예시:
- "content-digest는 요약을 **한국어로만** 안 해줌 (영어도 나옴)"
- "doc-coauthoring은 **우리 회사 템플릿**에 맞춰 안 써줌"
- "internal-comms는 **존댓말 강제**가 안 됨"

이 한 줄이 **Step 3 가공의 타깃**이 됩니다.
