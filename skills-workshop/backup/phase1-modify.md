# Phase 1 Step 3 — 가공(Modification) (15분) — **핵심 블록**

**목표**: 남의 스킬을 **내 것으로** 바꿔보는 성공 경험 1개. 이게 Phase 1 전체의 하이라이트.

**AskUserQuestion 예외 허용**: 이 Step은 Phase A에서도 AQ 사용 가능 (가공 방향 수집용).

---

## EXPLAIN

### 왜 "가공"이 중요한가

**0에서 1을 만드는 것보다, 1에서 1.1로 바꾸는 게 훨씬 쉬워요.**

개발자들이 오픈소스를 쓰는 방식이 이래요 — 남의 걸 가져와서 내 상황에 맞게 조금 고치는 거. 오늘 이걸 몸으로 해볼 겁니다.

### 가공 3가지 레벨 (쉬운 것부터)

| 레벨 | 방법 | 시간 |
|---|---|---|
| 🟢 Easy | description만 수정 (내 말투로) | 3분 |
| 🟡 Medium | body에 제약 추가 ("3줄 이내", "존댓말로" 등) | 5분 |
| 🔴 Hard | body의 Step 1개 추가/삭제 | 10분+ |

**오늘은 🟢 + 🟡 조합**으로 갑니다. 🔴는 필요하면 나중에.

### 원본은 건드리지 않는다 — `my-` prefix

원본을 망치면 나중에 되돌릴 수 없어요. 반드시 **복사본을 만들어서** 작업합니다. 복사본 이름 규칙: **`my-`** prefix (예: `my-content-digest`).

---

## EXECUTE

### Step 3-1. 가공 방향 선택 (2분, AQ 사용)

Phase 1 Step 2에서 찾은 "이 스킬이 안 해주는 것" 기억나시죠? 그걸 **어떻게 고칠지** 선택합니다.

```json
AskUserQuestion({
  "questions": [{
    "question": "이 스킬을 어떻게 '내 것'으로 만들고 싶으세요? (하나 골라주세요)",
    "header": "가공 방향",
    "options": [
      {"label": "내 말투 트리거 추가 (Easy)", "description": "description에 내가 실제로 쓸 법한 말 추가 (예: '나 훑어봐줘')"},
      {"label": "형식 제약 추가 (Medium)", "description": "'3줄 이내', '존댓말로', '표 출력' 같은 규칙 body에 추가"},
      {"label": "출력 포맷 변경 (Medium)", "description": "긴 요약→한 줄 / 표→리스트 같은 형식 변환"},
      {"label": "내 회사 맥락 반영 (Medium)", "description": "'데이원컴퍼니 템플릿으로', 'HR 관점으로' 같은 도메인 조정"}
    ],
    "multiSelect": false
  }]
})
```

### Step 3-2. 복사본 만들기 (2분)

Claude에게 요청:

```
[원본 스킬 이름]의 SKILL.md를 
C:\Users\admin\.claude\skills\my-[원본이름]\SKILL.md 로 복사해줘.
```

Claude가 Read로 원본 읽고, Write로 `my-` prefix 폴더에 저장합니다.

### Step 3-3. description 수정 (3분)

Claude에게 요청 (위에서 선택한 방향에 따라 문구 바꿔서):

```
방금 복사한 my-[원본이름]의 description을 고쳐줘:
- 원래 트리거 키워드는 유지하되, "[내 말투 추가]" 같은 표현도 발동되게 넣어줘
- 나는 [내 직군]이야
```

Claude가 Edit으로 수정한 SKILL.md를 보여줌. 확인.

### Step 3-4. body 수정 (6분)

`my-` 복사본의 body를 Edit 도구로 수정. Claude에게:

```
my-[원본이름]의 body에 아래 제약/변경을 추가해줘:
[Step 3-1에서 선택한 방향 구체화]

예시:
- "모든 출력은 3줄 이내로"
- "응답 문체는 존댓말로"  
- "표 대신 불릿 리스트로"
- "HR 직군 용어로 바꿔서"
```

Claude가 body 해당 섹션을 Edit으로 수정. 변경 전후 diff 확인.

### Step 3-5. 가공 전/후 비교 실행 (2분) — **감탄 순간**

같은 input으로 **원본**과 **가공본** 둘 다 돌려보세요.

```
[원본 이름]으로 이것 처리해줘: [샘플 input]
```
↓ 결과 확인
```
my-[원본이름]으로 이것 처리해줘: [같은 input]
```

차이를 **눈으로** 보세요. **"어? 진짜 내 것으로 바뀌었네"** 순간이 오면 Phase 1 완료.

```

```
