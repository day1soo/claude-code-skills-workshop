🔴 치명적 데모 스킬 3종이 npx skills add로 실제 설치 가능한지 미검증
문제
content-digest, internal-comms, doc-coauthoring 3종을 npx skills add [이름]로 설치하라고 안내. 하지만 이 스킬들이 공개 스킬 레지스트리(npx skills 지원 대상)에 등록돼 있는지 확인되지 않음 — 로컬 설치 스킬일 가능성 있음.
왜 문제인가
참가자 환경은 강사의 CLAUDE 환경과 다름. 강사 화면에서 데모 스킬이 보인다고 해서 참가자가 npx skills add content-digest 쳤을 때 똑같이 설치된다는 보장 없음. 설치 실패 시 Phase 1 Step 1 첫 5분에서 전원 멈춤 → 하이브리드 운영 붕괴.
수정 방향
파일럿 전 필수 검증: (1) 강사 본인이 아닌 새 환경에서 3종 스킬 설치 테스트. (2) 공개 스킬이 아닌 걸로 판명되면 — 대안 a) 사내 배포 가능한 공유 경로 마련 b) claude.ai/skills 공개 스킬 중 비개발자 친화적 1개 대체 선정 c) 참가자가 강사 머신의 스킬 폴더를 파일 복사로 받는 오프라인 경로 추가. 어느 쪽이든 recommended-skills.md 상단에 "배포 경로: ___" 명시.

🟢 제안 "body 4가지 질문 한 줄로 적어주세요" — 어디에 적는지 불명확
문제
EXECUTE Step 2-3에 "각 한 줄로 적어주세요 (머릿속 정리용)" 안내. 머릿속? 종이? Claude에게 보내기?
왜 문제인가

수정 방향
"Claude 터미널에 대화 메시지로 답해주세요" 명시 또는 "포스트잇에 한 줄씩 적어두세요. Step 3 가공할 때 씁니다" 같이 용도·장소 구체화. 강사가 배포하는 A6 힌트카드 뒷면에 체크박스로 미리 인쇄해두면 더 좋음.

phase1-modify.md (가공, 15분) — 핵심 블록

🔴 1 🟡 2 🟢 1
▼
🔴 치명적 my- prefix 복사본의 트리거 충돌 회피 안내 누락
문제
Step 3-2에서 원본을 my-[원본이름]으로 복사 → 원본·가공본이 같은 Claude 환경에 공존. description이 비슷하면 Claude가 "요약해줘" 입력 시 둘 중 어느 걸 꺼낼지 예측 불가. Step 3-5 "가공 전/후 비교"에서 사용자가 의도한 스킬과 다른 스킬이 발동할 가능성이 높음.
왜 문제인가
이 블록의 핵심 가치는 "가공 전/후 눈으로 차이 확인"으로 "어 다르네" 감탄 순간을 만드는 것. 그런데 트리거 충돌로 가공본이 안 꺼내지면 참가자는 "같은 결과가 나왔네, 가공이 의미 없었구나"로 잘못 해석 → 오늘 워크숍의 핵심 체험이 무너짐. 루브릭의 "실습 물리적으로 안 돌아감" = 🔴 치명적 기준에 해당.
수정 방향
Step 3-3 description 수정 파트에 "원본과 구분되는 트리거 1개 이상 필수" 규칙 추가. 예: "원본이 '요약해줘'에 반응하니, 가공본은 '내 스타일로 요약'이나 'ko 요약' 처럼 원본에 없는 단어를 반드시 포함". 그리고 Step 3-5 실행 시 원본 호출 프롬프트와 가공본 호출 프롬프트를 명시적으로 다르게 제시 (예: "요약해줘" vs "ko로 요약해줘").
🟡 개선 필요 Step 3-4 body 수정 — 선택한 방향을 실제 프롬프트 문구로 변환하는 템플릿 없음
문제
Step 3-1 AQ에서 "내 말투 트리거" 같은 추상 카테고리를 선택하게 함. Step 3-4에선 "[Step 3-1에서 선택한 방향 구체화]"를 Claude에게 입력하라고 함. 추상 카테고리 → 구체 문구 변환을 참가자가 혼자 해야 함.
왜 문제인가
비개발자가 "제약 추가" 선택했을 때, 실제로 뭐라고 입력해야 할지 막막함. 예시가 "3줄 이내", "존댓말로", "한국어만" 3개 있지만 본인 업무와 매칭 시키는 변환이 번거로움 → 시간 지연. Step 3-4가 6분 예산인데 입력 고민하다 끝남.
수정 방향
Step 3-1 각 선택지 옆에 입력할 프롬프트 완성본 연결. 예: "형식 제약 추가" 선택 → "아래 프롬프트 복사해서 본인 상황 단어만 바꾸세요: body에 '응답은 항상 ___ 이내', '문체는 ___'로 제약을 추가해줘". 변환이 아닌 빈칸 채우기로.
🟡 개선 필요 Step 3-2 복사 시 my-[원본이름] 폴더 생성 누락 가능
문제
"Claude가 Read로 원본 읽고, Write로 my- prefix 폴더에 저장합니다"로 기술. Write 도구가 없는 폴더를 자동 생성하긴 하지만, 환경에 따라 permission issue 발생 가능. 또한 Claude가 상위 폴더를 못 찾으면 다른 위치에 저장할 가능성.
왜 문제인가
복사 실패 시 Step 3-3 description 수정 대상이 없어서 참가자가 당황. 특히 원본 스킬이 .claude/plugins/ 경로에 설치돼 있을 경우, .claude/skills/로 복사하는 경로 해석 실패 가능성.
수정 방향
Step 3-2 실행 프롬프트를 명시적으로 2줄화: "먼저 C:\Users\admin\.claude\skills\my-[이름] 폴더를 만들어줘. 그 다음 원본 SKILL.md를 거기에 복사해줘." 또는 Bash mkdir -p && cp 직접 실행 안내. 그리고 복사 후 "ls .claude/skills/my-[이름]로 파일이 제대로 들어갔는지 확인" 체크포인트 추가.
🟢 제안 Step 3-5 가공 전/후 비교 — 실행 순서·동시 호출 애매
문제
"같은 input을 원본·가공본에 각각 넣고 차이 체험"이라 적혀있지만, 한 Claude 세션에서 두 스킬을 연달아 호출하는 구체 흐름(세션 분리? 새 대화?)이 모호.
왜 문제인가
비개발자가 같은 대화창에서 원본 호출 → 결과 확인 → 가공본 호출 → 결과 비교할 때, Claude가 이전 응답 맥락에 오염돼서 두 번째 응답이 첫 번째와 동화될 가능성. 차이가 흐려져 감탄 순간이 약화.
수정 방향
"원본 실행 → 결과 눈으로 확인 → 새 대화 시작 → 같은 input으로 가공본 실행" 순서 명시. 또는 원본·가공본 호출 프롬프트 사이에 "이전 결과는 잊고" 한 줄 넣기.

phase2-create.md (내 스킬 제작, 15분)

🟡 개선 필요 Step 1-1 업무 옵션 4개가 HR·운영 편향 (실무-실습 갭 안티패턴 #2)

전사 대상 워크숍에서 마케팅·영업 직군 참가자가 "Other"를 누르게 됨 → AQ 질문의 기본 가정이 "다른 직군은 예외"로 전달. 불편함. 또한 첫 선택에서 본인 직군이 안 보이면 "이 워크숍은 내 일에 안 맞나" 방어적 태도 유발.
수정 방향
옵션 4개를 직군 분포에 맞춰 재구성. 예: "주간보고 / 회의 액션아이템 / 데이터 요약·리포트 / 외부 커뮤니케이션 초안" — 4개로도 커버 폭 확장 가능. 또는 강사가 사전 참가자 분포를 받고 recommended-skills.md처럼 선정 결정 트리로 옵션을 당일 아침 세팅.

templates/backup-scenarios.md 신설 — 직군별(HR/영업/마케팅/운영) 각 1개씩, 가상 Input/Output 세트 완성본. 강사가 당일 아침 인쇄해서 미지참자에게 즉시 지급. 카드 내용에는 "이 시나리오는 예시이니 본인 업무와 가장 비슷한 걸로 진행하세요" 안내 포함.
